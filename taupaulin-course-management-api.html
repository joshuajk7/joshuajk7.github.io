Tarpaulin Course Management API

← Back to Portfolio

Description

The Tarpaulin API is a secure and scalable course management backend system, developed as a cloud-native RESTful service using Python (Flask), Google App Engine, and Google Cloud Datastore. The project models a lightweight alternative to traditional LMS systems like Canvas and supports user roles such as admin, instructor, and student.

Key Features

  • 13 RESTful API endpoints supporting CRUD operations for users, courses, and enrollment data
  • Role-based access control using JWT and Auth0 for secure authentication and authorization
  • Avatar upload and download functionality integrated with Google Cloud Storage
  • Datastore modeling for users, roles, courses, and relationships using advanced filtering and pagination
  • Comprehensive error handling for common HTTP error responses (401, 403, 404, etc.)
  • Deployed and maintained on Google App Engine for high availability and scalability

Technologies Used

  • Python 3
  • Flask
  • Google Cloud Platform (App Engine, Datastore, Cloud Storage)
  • Auth0 (OAuth 2.0 / JWT)
  • Postman

Security & Architecture Highlights

  • Only authorized users can perform actions based on their assigned roles (e.g., students cannot manage courses).
  • All endpoints validate tokens using public keys from Auth0's JWKS endpoint, preventing forgery.
  • Avatar uploads use secure signed URLs and are validated before being saved in cloud storage.
  • All sensitive operations are protected with scoped permissions and granular checks at the route level.

  • Implementing RBAC (Role-Based Access Control) and protecting endpoints with JWT verification
  • Designing scalable datastore schemas and relationships in a NoSQL context
  • Working with Google Cloud APIs programmatically (Datastore, Storage)
