Description

This full-stack database application was designed for Global Bread Co., a company aiming to generate $500,000 in revenue from over 1,000 sales across 60+ bread products with global cultural origins. The platform allows administrators to manage customers, bread products, sales, allergens, and regional cultures through a secure web interface.

A major design goal was ensuring accurate tracking of allergens and product origins. Two intersection tables, soldProducts and allergensProducts, were critical to this functionality and featured prominently in the “Sales” and “Allergens” pages respectively.

Key backend optimizations included converting all decimal attributes to decimal(10,2) to improve currency formatting, and adding ON DELETE SET NULL to the productID foreign key in soldProducts to preserve historical sales data even if a product was deleted.

The frontend UI features edit pages with dynamic dropdowns for related foreign keys and auto-calculation of line totals for each sale. The interface was styled using Bootstrap and custom CSS with warm, welcoming tones to reflect the brand identity.

Significant time was spent troubleshooting database-server connections, ultimately resolved by adding a helper function that establishes a connection before executing queries.
